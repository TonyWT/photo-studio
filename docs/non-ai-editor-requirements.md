# Photo Studio 非 AI 编辑器完整需求与验收清单

> 范围：基于 Pixlr Express 的桌面编辑工作流，只保留通用手工编辑能力。本文是实现顺序和验收的唯一清单；不得把 miniPaint 已有但未接入工作台的能力标成完成。
>
> 明确排除：AI Tools、AI Cutout、Element、在线素材/图片搜索、远程模型推理、在线字体与账号/云同步。
>
> 侧栏所有可见控件的编号、截图来源和验收粒度见 [`pixlr-express-side-panel-checklist.md`](./pixlr-express-side-panel-checklist.md)；本文件保留跨模块不变量和高层门槛。

## 全局不变量

| ID | 要求 | 验收证据 |
| --- | --- | --- |
| G-01 | 所有图片只在浏览器本地处理；导入只接收本地文件。 | 网络拦截/源码测试，无远程图片或推理请求。 |
| G-02 | 每个改变画布或图层的操作都进入历史栈，底栏可撤销/重做。 | E2E 在每类工具至少覆盖一次“操作→撤销→重做”。 |
| G-03 | 锁定的图层不能被移动、像素修改、裁剪内修改或手工抠图修改。 | E2E 锁定后尝试编辑，图层状态和导出未改变。 |
| G-04 | 工作台只显示非 AI 工具；AI/Element 既无 DOM 入口，也无快捷键或网络路径。 | DOM、源码与请求测试。 |
| G-05 | 所有本地项目可自动暂存、手动保存、从首页恢复；导出不依赖登录。 | IndexedDB + 下载 E2E。 |

## 完成判定

“功能可用；视觉 P2”表示操作、数据、撤销与自动化证据已完成，但仍不能宣称“完整还原”。只有全部保留项均具备功能证据，且工作台在固定 1920×878/已加载图片状态下没有可行动的 P0/P1/P2 视觉差异时，才能作出该结论。

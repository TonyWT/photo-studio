# Pixlr Express 非 AI 功能还原矩阵

> 参考取证：2026-07-13，Pixlr Express 桌面编辑器 `1920 × 878`、已打开栅格图片状态。只还原通用布局、交互层级和非 AI 能力；不复制 Pixlr 品牌、文案、插画、用户头像或素材。

> 控件级验收清单见 [`pixlr-express-side-panel-checklist.md`](./pixlr-express-side-panel-checklist.md)。高层 A-* 条目不能替代该清单中的单个按钮/参数/卡片验收。

状态定义：`可复用` 表示 miniPaint 已有内核能力但尚未进入 Pixlr 式壳层；`缺失` 表示需要新增实现；`排除` 为用户明确不要、不得显示或请求的能力。

## 3. 完整性门槛

- 每个 `A-*` 条目必须从工具轨或 Home 到达，且有一个真实操作结果和撤销/重做证据。
- 每个 `X-*` 条目必须没有 UI 入口、快捷键或请求路径。
- `W-*` 必须与同尺寸参考截图进行视觉比对；P0/P1/P2 差异全部修复才可标记工作台完成。
- 未满足的条目必须保持“缺失/部分可用”，不得用“内核存在”替代验收。

# Pixlr Express 侧边栏逐项需求清单

> 来源：用户于 2026-07-14 提供的 9 张桌面端编辑器截图（Crop、Cutout、Adjust、Effect、Filter、Liquify、Retouch、Draw、Text）。
>
> 目标：不再只以“工具已接通”作为完成标准。每个侧栏中的按钮、滑杆、开关、卡片、Apply/Cancel 行为都必须拥有独立编号、实施状态和自动化验收。此清单是 [`pixlr-express-parity-matrix.md`](./pixlr-express-parity-matrix.md) 的控件级展开。

## 判定规则

- **保留**：必须有可到达入口、真实本地处理结果、撤销/重做，以及对应自动化证据。
- **排除**：必须没有入口、快捷键和网络/模型请求；不是“暂未实现”。
- **待取证**：截图仅显示图标或未展示滚动区域，不能凭图标猜测功能；取到名称、交互和结果后才能转为保留或排除。
- **本地实现**：不复制 Pixlr 品牌、文案、图库图片、文字海报素材或任何在线资源。预设只复刻可编辑的功能层级，缩略图使用原创本地测试/预设资产。
- **Smart Resize**：在确认其是否模型/远程能力前不得实现；若属于 AI/远程推理则归入排除，否则才列入 Crop 的本地图像尺寸功能。

### 本轮截图的范围结论

是，九个侧栏都进入需求列表；但清单不是把每个看得见的图标都机械地承诺为“照抄”。每项都先落到以下三类之一：

| 类别 | 处理方式 | 本轮截图中的例子 |
| --- | --- | --- |
| 本地保留 | 做成真实、可撤销、可导出的浏览器本地操作，并逐项测试 | Crop、手动 Cutout、Adjust、Effect、Filter、Liquify、Retouch、Draw、Text |
| 本地替代 | 保留编辑能力和交互层级，但使用原创本地预设/缩略图，不复制 Pixlr 的素材、品牌文案或成品海报 | Effect 分类卡、Text 样式卡、Draw 笔刷预设 |
| 明确排除 | 从 UI、快捷键、网络请求中都不存在，并在清单中说明原因 | AI Cutout、AI Smart Resize、Object Healing、Element |

因此，截图中每个按钮、滑杆、开关、预设卡和 Apply/Cancel 都会有单独编号；截图未能确认语义的图标先标为“待取证”，不会用空按钮凑数。

## 范围锁定：侧栏不是“一级菜单清单”

本清单以**每个可见、可操作控件**为最小需求单位，而不是只登记左侧的九个一级入口。下列项目全部纳入交付与验收范围：

| 一级入口 | 必须逐项登记的二级能力 | 明确排除或待技术判定 |
| --- | --- | --- |
| Crop | 宽高、拉直、比例、比例锁定、旋转/翻转、Image size、Canvas size、Cancel/Apply | Smart resize 先判定是否为本地确定性算法 |
| Cutout | 四种手动工具、Keep/Remove、六种形状、Softness、提示、反选、重置、应用 | **AI Cutout 排除** |
| Adjust | 三个快捷操作、Color/Light/Details/Scene 的每个滑杆、重置/预览/应用 | 无 |
| Effect | 每个分类卡、分类中的每一个效果预设、强度、返回、Cancel/Apply | 无；不得复制 Pixlr 的分类图片或文案资产 |
| Filter | 每张滤镜卡、参数、预览、Cancel/Apply | 无 |
| Liquify | 六个变形笔刷、Size/Strength/Density、HQ 预览、Close | 无 |
| Retouch | 四个工具、Spot/Object、Size、每个修复子工具、Close | Object 先判定是否依赖 AI/远程；若是则排除 |
| Draw | 五个工具、颜色、笔刷、Size/Softness/Transparency、色样、笔刷形状、Close | 无 |
| Text | 新建文字、默认文字、每一张可编辑样式卡、滚动续项 | 不复制 Pixlr 的现成文字海报/素材 |

因此，“入口已能打开”不构成完成。任何一个子按钮、滑杆、开关、预设卡或确认动作仍是独立需求；有未取证、未实现或无自动化证据的子项时，对应一级入口只能标为“部分可用”。

## 与高层矩阵的关系

- A-03 对应 C-*；A-04 对应 K-*；A-05 对应 J-*；A-06 对应 E-*；A-07 对应 F-*；A-08 对应 L-*；A-09 对应 R-*；A-10 对应 D-*；A-11 对应 T-*。

- `待取证` 不是完成状态；它会阻止相应高层条目被标记为“完整还原”。
